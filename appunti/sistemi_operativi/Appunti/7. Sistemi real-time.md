# Sistemi real-time
>[!important] Sistemi real-time
>È un sistema che richiede che un insieme di operazioni siano prodotti entro un determinato intervallo temporale.

>[!important] Sistema embedded
>È un dispositivo di elaborazione che è inserito in un sistema più grande e più complesso.

>[!important] Sistemi safety-critical
>È un sistema real-time che in caso di guasto/fallimento può causare risultati catastrofici

![[0. Intro sistemi operativi#Sistemi real-time]]
![[1. Gestione dei processi#Scheduling real-time]]

I sistemi hard real time hanno dei vincoli temporali strettissimi, mentre i soft real time hanno delle priorità diverse.

Tipicamente i sistemi real-time possiedono le seguenti caratteristiche:
- **single purpose**
- **small size**
- sono **economici** se prodotti in grandi quantità
- specificano requisiti stretti di **timing**

>[!important] System on a chip (SOC)
>Molti sistemi real time sono realizzati utilizzando particolari architetture **SOC** che permettono di inserire CPU, memoria, MMU e periferiche sullo stesso circuito integrato.

Per l'implementazione software di un sistema real-time è necessario un kernel che spesso è diverso dai tradizionali, in quanto:
- sono single purpose
- non hanno bisogno di interfacce utente
- devono gestire meno dispositivi dei pc

## Scheduling del processi periodici
Nei sistemi hard real-time i processi periodici richiedono risorse alla CPU a intervalli di tempo periodici chiamati **periodi**.
Una volta ottenuto l'accesso alla CPU un processo periodico viene eseguito per un periodo di tempo fissato tale per cui
$$
0 < t \leq d \leq p
$$
dove:
- $p$ **periodo**
- $d$ **scadenza**
- $t$ **tempo** di elaborazione
La differenza tra periodo e scadenza rappresenta la **tolleranza** del sistema.
